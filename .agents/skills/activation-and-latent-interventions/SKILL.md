# Activation and Latent Interventions

## When to Use

- Use this skill when designing white-box or open-weight experiments that identify, steer, ablate, or transfer latent features.
- Use it for causal tests of authority, refusal, policy, reasoning, or behavior features.
- Do not use it as a shortcut for deploying unvalidated model edits.

## Inputs

- Model with hidden-state access.
- Contrastive datasets or labeled feature examples.
- Intervention surface: residual stream, attention output, MLP output, SAE latent, token embedding, steering token, vector field, or KV cache.
- Behavioral and geometry metrics.

## Core Model

- Representation engineering treats high-level behavior as measurable directions, subspaces, features, or operators in activation space.
- Simple activation addition can work, but the corpus suggests stronger variants: contrastive activation addition, mean-centered vectors, conceptors, SAE-targeted features, compositional steering tokens, vector fields, and KV-cache edits.
- Causality requires intervention. Correlation, probe accuracy, or visual clustering is not enough.
- Latent interventions may be non-surjective: the steered activation can fall outside states reachable by any ordinary prompt. Interpret text-prompt analogies carefully.

## Workflow

1. Define the behavior and its safe measurement target.
2. Build contrastive pairs or labeled examples that isolate the feature from topic, sentiment, length, and style.
3. Identify candidate latent objects:
   - difference-in-means vector
   - logistic probe normal
   - PCA or low-rank subspace
   - conceptor matrix
   - SAE feature or sparse feature set
   - context-conditioned vector field
   - learned steering/control token
   - KV-cache memory or patch
4. Localize by layer, position, and generation phase.
5. Intervene with sweeped strengths and controls: add, subtract, project out, clamp, compose, or route through a learned operator.
6. Measure geometry and behavior together. Track off-target degradation and reversibility.
7. Compare against prompt-only baselines and negative-control features.

## Design Checks

- Mean-center when comparing activation additions across prompts with different base distributions.
- Use held-out topics and paraphrases to avoid learning prompt artifacts.
- For SAE work, report feature sparsity, reconstruction quality, and whether targeted features outperform dense vectors.
- For vector fields, condition intervention on context rather than applying a global vector blindly.
- For KV-cache work, treat cached states as sensitive and model/version-specific.

## Metrics

- Target behavior score and dose-response curve.
- Off-target utility: perplexity, benchmark loss, fluency, instruction following.
- Geometry: cosine, projection magnitude, probe score, feature activation, reconstruction error.
- Locality: layer/position/token sensitivity.
- Robustness: paraphrase, sampling, prompt length, template, and model transfer.

## Outputs

- An intervention card with dataset, latent object, layer/position, strength sweep, controls, and metrics.
- A causal claim scoped to the measured model and behavior.
- A list of failure modes and suspected confounds.

## Constraints

- Do not infer deployable safety changes from one open-weight model without transfer and utility checks.
- Do not publish harmful completions generated by safety-suppression interventions.
- Verify corrupted formulas against source PDFs before implementing math-sensitive operators.

## Verification

- Include no-intervention, random-vector, opposite-direction, and shuffled-label controls.
- Show effect-size curves, not just cherry-picked generations.
- Re-run the best setting on held-out prompts and at least one neighboring layer.
